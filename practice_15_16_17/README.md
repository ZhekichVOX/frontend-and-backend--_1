- HTTPS-инструкции и App Shell архитектура (`index.html` + динамический `content/*`).

## Особенность: Откладывание уведомлений (Snooze)

Когда приходит push-уведомление, пользователь может выбрать одно из трёх действий откладывания:

- **⏰ 5 мин** - получит напоминание через 5 минут
- **⏰ 10 мин** - получит напоминание через 10 минут
- **⏰ 15 мин** - получит напоминание через 15 минут

### Как это работает

1. **Service Worker (sw.js)** показывает 4 кнопки действий в уведомлении
2. Когда пользователь нажимает кнопку "⏰ X мин", Service Worker:
   - Отправляет запрос на сервер `/schedule-push`
   - Передаёт текст задачи и время откладывания
   - Закрывает текущее уведомление
3. **Сервер (server.js)** получает запрос и:
   - Устанавливает таймер на указанное время
   - Через N минут отправляет повторное push-уведомление
   - Логирует всё в консоль для отладки

## Тестирование Snooze функции

1. **Откройте приложение** и **включите уведомления**

2. **Добавьте новую заметку**

3. **Появится уведомление с кнопками:**
   ```

   [⏰ 5 мин] [⏰ 10 мин] [⏰ 15 мин] [✓ Открыть]
   ```

4. **Нажмите одну из кнопок откладывания**

5. **Проверьте в консоли браузера:**
   ```
   ⏰ Отложено на 5 минут
   ✅ Уведомление отложено на 0.08333333 минут
   ```

6. **Спустя указанное время получите повторное уведомление:**
   ```
   ⏰ Напоминание задачи: Мой текст
   ```

## API Endpoints

```
POST /subscribe                 - Подписаться на push-уведомления
POST /unsubscribe               - Отписаться от push-уведомлений
POST /schedule-push             - Отложить уведомление
GET /api/subscriptions-count    - Проверить количество подписок и отложенных уведомлений
```

### Endpoint /schedule-push

**Запрос:**
```json
POST /schedule-push
Content-Type: application/json

{
  "text": "Текст задачи",
  "delay": 300000,  // в миллисекундах (300000 = 5 минут)
  "id": "1234567890"
}
```

**Ответ:**
```json
{
  "success": true,
  "message": "Уведомление отложено на 5 минут",
  "scheduledIn": "5 мин"
}
```

## Возможные проблемы

### Push-уведомления не приходят?

Смотрите PUSH_NOTIFICATIONS_REPORT.md в корне проекта

### Отложенные уведомления не приходят?

1. **Проверьте консоль сервера** - должны видеть логи `⏰ Отложено`
2. **Проверьте, что браузер открыт** - Service Worker работает только если вкладка активна
3. **Проверьте консоль браузера** на ошибки сети

### Несколько таймеров одновременно?

Если пользователь нажимает несколько кнопок откладывания подряд:
- Автоматически отменяется предыдущий таймер (по ID)
- Остаётся только последний таймер

## Особенности реализации

✅ Таймеры хранятся на сервере (в памяти)  
✅ При перезагрузке сервера отложенные уведомления теряются  
✅ Если подписка удалена, её отложенные уведомления также удаляются  
✅ Каждое отложенное уведомление имеет уникальный ID  
✅ Текст отложенного уведомления содержит префикс "⏰ Напоминание"
